# 日次：ITパスポート 問題追加プロンプト

このファイルは、日次のタスクスケジューラ（cron / GitHub Actions / Claude Code の `/loop` 等）から呼び出すことを想定したプロンプトです。
実行するたびに、`app.js` の `QUESTIONS` 配列へ **新しい問題をできるだけ多く（推奨: 10〜20問）追加** します。

---

## プロンプト本文（そのままコピーして使用可）

```text
あなたはITパスポート試験の出題者です。
リポジトリ直下の `app.js` に含まれる `QUESTIONS` 配列へ、新しい問題を **できるだけ多く追加** してください（推奨: 10〜20問）。

## 事前に読むべきファイル
- `app.js` — 既存の `QUESTIONS` / `TERMS` 定数の形式と、すでに登録済みの問題を必ず確認すること

## データスキーマ（厳守）
既存の `QUESTIONS` の要素と完全に同じ形式で追加すること。

```js
{
  id: <次の連番>,
  category: "ストラテジ" | "マネジメント" | "テクノロジ",
  year: "令和X年Y月",     // 実在する出題回（令和3年〜令和6年のいずれか、月は4月または10月）
  question: "…どれか。", // 1文、敬体は使わず試験文体で
  choices: [
    "選択肢1",            // ア/イ/ウ/エ のラベルは付けないこと（ランタイムで自動付与される）
    "選択肢2",
    "選択肢3",
    "選択肢4"
  ],
  answerIndex: 0,         // 0〜3。必ず毎問ランダムに分散させ、同じ値ばかりにしないこと
  explanation: "…"       // 2〜3文で簡潔に
}
```

## 追加ルール（すべて必須）
1. **`id` は既存の最大値 + 1 から連番で付与**すること。
2. **重複禁止**：既存の `question` と意味的に重複する問題は作らない。`category` と主題（キーワード）で類似度を確認すること。
3. **分野バランス**：追加する複数問は「ストラテジ」「マネジメント」「テクノロジ」が偏らないよう、直近 20〜30 問程度の分布を見ながら調整する。理想は 3 分野がおおむね均等な割合で、不足分野を優先。
4. **出題根拠**：実際の「ITパスポート試験 過去問題（過去問道場）」相当の実在する典型問題を参照し、捏造的な用語や存在しない規格を含めない。
5. **選択肢の長さ**：4 選択肢は概ね同程度の長さに整え、正解だけが極端に長い/短いなどの手がかりを与えない。
6. **正解位置の分散**：`answerIndex` は 0〜3 がなるべく均等に分散するように、連続して同じ値が 3 回以上続かないようにすること。
7. **表示側ではア/イ/ウ/エの順序は実行時にシャッフル**されるので、選択肢の順序自体に意味は持たせない。
8. **`explanation`** は「なぜそれが正解なのか」を端的に述べ、他の選択肢がなぜ誤りかを軽く触れる。

## 編集手順
1. `app.js` を読み、既存の `QUESTIONS` の末尾要素の `id` を確認する。
2. `QUESTIONS` 配列の **末尾** に新しい 3 問を追記する（既存の整形スタイル・インデントを厳守）。
3. 他の関数、`TERMS` 配列、HTML/CSS には手を加えない。
4. 以下を実行し、構文エラーがないことを確認する：
   ```bash
   node -e "new Function(require('fs').readFileSync('app.js','utf8'))"
   ```
5. `git add app.js` し、`git commit -m "Add multiple IT Passport questions (Nサンプル問題)"` でコミットする。（N は追加問題数）
6. `git push -u origin claude/it-passport-quiz-app-1nmqQ` でプッシュする。
   - もし別ブランチで運用している場合はそのブランチへプッシュする。

## 出力
- 作業完了後、「追加した全問題の id / category / 一行要約」を箇条書きで報告する。
- 既存問題と重複していた場合は、その旨と差し替えた問題を報告する。
- 最後に追加問題数と各分野の内訳（ストラテジ X 問、マネジメント Y 問、テクノロジ Z 問）をサマリーで示す。
```

## 日次スケジューラでの実行例

### Claude Code のループスキルで実行
```bash
/loop 24h /bash claude --print "$(cat prompts/add-questions.md | sed -n '/## プロンプト本文/,/^---$/p')"
```

### cron から Claude CLI を呼ぶ
```cron
# 毎日 09:00 に実行
0 9 * * * cd /path/to/ipasu && claude --print "$(cat prompts/add-questions.md)" >> /var/log/ipasu-add-questions.log 2>&1
```

### GitHub Actions（daily schedule）
`.github/workflows/add-questions.yml` を用意し、`schedule: - cron: '0 0 * * *'` でトリガー、Anthropic API 経由でこのプロンプトを送る構成にする（API キーはリポジトリシークレットに設定）。

---

## 注意
- 問題数が増えすぎた場合はスケジューラの頻度を下げるか、問題セットを複数ファイルに分割することを検討する。
- 実行ログは必ず保存し、重複や誤答の混入を週次で目視レビューすることを推奨する。
